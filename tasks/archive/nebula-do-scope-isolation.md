## Objective

Make `NebulaDO`'s tenant isolation **structural** — a mesh call is accepted only if its `aud` is covered by the scope encoded in the DO's **instance name** — replacing the **trust-on-first-use (TOFU)** `aud`-lock. This closes **Star pre-claim** and **shared-Galaxy/Universe multi-star rejection** (the demo blocker).

## Background — what we found (verified in code)

`NebulaDO.onBeforeCall` ([nebula-do.ts:30-46](../apps/nebula/src/nebula-do.ts#L30)) stores the **first** caller's `aud` in `__nebula_universeGalaxyStarId` and rejects later mismatches; it **never compares `aud` to `this.lmz.instanceName`**.

**`onBeforeCall` is the SOLE tenant gate on the mesh hot path — not a redundant backstop.** The Gateway forwards the **client-chosen `(binding, instance)` verbatim** — `NebulaClientGateway` overrides only `onBeforeCallToClient` (the push direction), there is no inbound scope check ([nebula-client-gateway.ts:13](../apps/nebula/src/nebula-client-gateway.ts#L13)), and `#handleClientCall` routes to `getDOStub(env[binding], instance)` with only `aud` pinned to the verified WS attachment. The HTTP-path `matchAccess` ([router.ts:382](../packages/nebula-auth/src/router.ts#L382)) does **not** run on per-call mesh dispatch. So a logged-in `aud=acme.app.tenant-a` client can issue `lmz.call('STAR','<victim>')` directly; only `onBeforeCall` stops it. (A foreign **DO** can do the same DO→DO; only the callee's own `onBeforeCall` covers both origins.)

Two consequences of TOFU-not-structural (both tier DOs):

1. **Star pre-claim.** Because `onBeforeCall` never checks `aud`-vs-name, an attacker addressing `STAR/<victim>` with their own `aud` *before the victim's first login* sets the TOFU lock to the wrong scope. (Severity-bounded — see *Why this is sound* below.)
2. **Shared Galaxy/Universe multi-star.** `Galaxy extends NebulaDO` (no override); it's called both by Stars with their inherited **star-level** `aud` ([star.ts:261-262, :343-344, :418-419](../apps/nebula/src/star.ts#L261)) and by galaxy-admins with **galaxy-level** `aud`. TOFU locks to whichever `aud`-level arrives **first** and rejects the rest — so a galaxy-shared ontology can't serve a second star. **Latent in tests** (unique universe per run masks it).
   - **Dev-Star bump (2026-06-16):** the dev-Star decision (`tasks/dev-star.md`) makes this concrete — `{u}.{g}.dev` and any production `{u}.{g}.{s}` share Galaxy DO `{u}.{g}` and both call `getLatestOntologyVersion` with star-level `aud`, so TOFU rejects whichever arrives second. Goes live the moment two Stars under one Galaxy are active together. **Confirmed a demo blocker → on the `nebula.md` critical path.**

A third class — tenant-scoped **helper** DOs named by something other than their scope — is the deferred **Fix 2** ([on-hold/nebula-scoped-helper-naming.md](on-hold/nebula-scoped-helper-naming.md)). No production helper exists; the `ResourceHistory` fixture is removed here (Phase 2).

Root: isolation rests on first-use luck + name-unguessability, not structure.

## Design — Fix 1: structural, tier-aware `onBeforeCall`

Accept iff `aud` is covered by the scope encoded in the instance name, reusing the existing id-algebra. For a tier DO the instance name **is** the scope (a 1–3-segment id), so there is no colon-split — the name goes straight into `buildAuthScopePattern`:

```ts
onBeforeCall() {
  const name = this.lmz.instanceName;
  if (!name) throw new Error('Mesh call missing callee instance name');     // (a) fail-closed — envelope lacked metadata.callee
  if (isPlatformInstance(name)) throw new Error('Active-scope mismatch');    // (b) platform guard — buildAuthScopePattern('nebula-platform')==='*' (accept-all)
  const pattern = buildAuthScopePattern(name);                              // (d) throws if name unparseable → fail-closed
  const aud = this.lmz.callContext.originAuth?.claims?.aud;
  if (!aud) throw new Error('Missing active scope (aud)');                   // (c) unchanged
  if (!matchAccess(pattern, aud)) throw new Error('Active-scope mismatch');  // (e) tenant boundary
}
```
- **All rejections are `Error` instances, never bare strings (B1).** A thrown string survives `preprocess` as a plain string ([preprocess.ts:225](../packages/structured-clone/src/preprocess.ts#L225)) and lands in `lastResult`, not `lastError` ([baseline/index.ts:206](../apps/nebula/test/test-apps/baseline/index.ts#L206)) — so every `lastError` assertion would time out RED in a confusing way. (The current code already uses `new Error`; keep it.)
- **Star** (name = star id) → exact pattern → `aud` must equal the star.
- **Galaxy / Universe** (name = galaxy / universe id) → `…​.*` → accepts the scope itself **and any descendant**, rejecting other tenants. This is the intended "a Star reaches its own Galaxy/Universe (and they it), never another's" model. **Authorization stays per-method:** the scope check is the *tenant boundary*; whether a call must be admin-originated is decided by `@mesh(requireAdmin)` as today, and admin-ness is **orthogonal** to the tenant boundary (a galaxy-A admin, `aud=acme-A.app`, is still rejected by `matchAccess` from reaching `GALAXY/acme-B.app`).
  - **Phase-1 audit + ongoing-invariant guard (M1 + B5):** the `.*` widening is sound **only** if every **non-admin** `@mesh` method on `Galaxy`/`Universe` holds galaxy/universe-**shared** data any descendant star may read. Current surface (`grep -n '@mesh()' apps/nebula/src/{galaxy,universe}.ts`): **Galaxy** `getGalaxyConfig` ([galaxy.ts:95](../apps/nebula/src/galaxy.ts#L95)), `getLatestOntologyVersion` (:133), `getOntologyVersion` (:144), `listOntologyVersions` (:150); **Universe** `getUniverseConfig` ([universe.ts:17](../apps/nebula/src/universe.ts#L17)) — **five** (decorator lines are one above each body). The three **admin** methods (`setGalaxyConfig`@87, `appendOntologyVersion`@104, `setUniverseConfig`@9) sit under the same widened boundary; their `@mesh(requireAdmin)` stays the authorization wall. `getLatestOntologyVersion` is load-bearing — Stars call it cross-tier with a **star-level** `aud` ([star.ts:261-262, :343-344, :418-419](../apps/nebula/src/star.ts#L261)), so its "shared, any descendant may read" classification is what proves the widening sound. **Enforce as an observable artifact, not prose (B5):** ship a test asserting the non-admin `@mesh` method set on Galaxy/Universe **equals a frozen allow-list** — adding a new non-admin method fails the test, forcing the author to classify it as shared-tenant data. **Derive the set dynamically** (walk the prototype, filter mesh-callable, partition by `getMeshGuard(m) !== requireAdmin` — `requireAdmin` is one identity-comparable import) — never a hand-typed literal, which would silently accept a new leaky method. This survives the task file freezing on archive.
- **Stop reading/writing the `__nebula_universeGalaxyStarId` store.** The new `onBeforeCall` replaces the TOFU read+write entirely (confirmed single writer/reader, nebula-do.ts:40-42). Existing stored values become inert — left in place, not deleted (see *Migration*).
- **Platform-name guard (M-1).** `buildAuthScopePattern('nebula-platform')` returns `'*'`, and `matchAccess('*', anyAud)` is always true — so a tenant DO bearing the platform instance name would collapse the gate to accept-all. None of Universe/Galaxy/Star is the platform DO, so `onBeforeCall` hard-rejects `isPlatformInstance(name)` first. Covered by **T-platform**.
- **Why this is sound (M4 — it *relocates* TOFU, doesn't "kill" it).** `this.lmz.instanceName` is itself first-write-wins (`setInstanceName`, [lmz-api.ts:588](../packages/mesh/src/lmz-api.ts#L588)). The invariant: a tier NebulaDO is **addressed by the same `parseId`-valid id that becomes its `instanceName`** — never a 64-char-hex DO ID (slugs aren't 64-hex), so `getDOStub`'s `idFromString` branch ([get-do-stub.ts:30-37](../packages/routing/src/get-do-stub.ts#L30)) is unreachable for these names — hence **derived scope == routing key**. An attacker can only write a name into the DO *at that name's address*; they can't make a victim's DO adopt a foreign name, and `matchAccess(buildAuthScopePattern(name), attackerAud)` fails closed. Pre-claim degrades to **benign DoS-by-naming**, not a hijack.
- **Stamping invariant (m-3).** Soundness requires `metadata.callee.instanceName` === the routing key passed to `getDOStub` on **every** path reaching a NebulaDO (Gateway `#handleClientCall`; `callRawImpl`) — name and address are the same string by construction, so `onBeforeCall` deriving scope from the name is equivalent to deriving it from the address. The runtime backstop is the **framework's** `setInstanceName` first-write-wins guard ([lmz-api.ts:588](../packages/mesh/src/lmz-api.ts#L588)), *not* `onBeforeCall` (which simply uses whatever name was stamped). T-stamp exercises that framework guard as defense-in-depth — it is **not** a Fix-1 `onBeforeCall` branch.
- **`instanceName` is readable here (M7).** `__init` runs from envelope metadata *before* `onBeforeCall` ([lmz-api.ts:877-895](../packages/mesh/src/lmz-api.ts#L877)) and persists — but only when `metadata.callee.instanceName` is present; absent ⇒ the fail-closed throw (a). (The alarm-delivered and OCAN self-continuation paths run via the *local* executor, not `executeEnvelope`, so this throw never fires on them — see T-local-skip.)
- **Dependency / placement.** Promote nebula-do.ts's existing type-only imports of `buildAuthScopePattern`, `matchAccess`, `isPlatformInstance` to **value** imports — all three already live in `@lumenize/nebula-auth` beside `parseId`. Direction is nebula→nebula-auth (no cycle). (The colon-helper relocation that would have raised a `routing→nebula-auth→routing` cycle was Fix-2-only; it's captured in the on-hold file, not a concern here.)

## Migration — no migration needed; the stale value is inert (M2)

The stored `__nebula_universeGalaxyStarId` is **whatever `aud` first claimed the DO** (nebula-do.ts:40-42) — per-first-caller and tier-ambiguous, hence structurally untrustworthy: "read-through-then-reject-on-mismatch" would both lock out legitimately-claimed instances and rest on a value that doesn't reliably encode tier. **Trust the instance name; the new `onBeforeCall` never reads or writes the stored value, so existing stored values are inert dead data — leave them in place.** No cleanup delete is needed (and one would be a footgun: the natural `onStart()` home is shadowed on the `Star`/`DevStar` tier, whose `Star.onStart()` doesn't call `super.onStart()` — [star.ts:42-60](../apps/nebula/src/star.ts#L42) — so a base-class delete would silently skip the very tier most likely to hold a stale key, for zero benefit since the value is never read). **Don't re-add a compare later** (the B5 allow-list test + T-migration guard against it).
