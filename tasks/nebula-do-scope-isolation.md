# Nebula DO Scope Isolation ‚Äî Structural, not Trust-On-First-Use

**Status**: Design ‚Äî **DO FIRST** (precedes [broadcast-origin-transparency.md](on-hold/broadcast-origin-transparency.md), which is on hold)
**Origin**: Surfaced during the broadcast design review (2026-06-08); hardened by a dedicated reviewer panel (workflow `w0v2nupbw`) that corrected several premises below, then re-reviewed 2026-06-08 by a second panel that added M-1 (platform-name accept-all), M-2 (full @mesh audit surface), M-3 (test-rewrite disposition), m-1 (single scope derivation), and m-4 (migration rationale).

## Objective

Make `NebulaDO`'s tenant isolation **structural** ‚Äî a mesh call is accepted only if its `aud` is covered by the scope encoded in the DO's **instance name** ‚Äî replacing the current **trust-on-first-use (TOFU)** `aud`-lock. Plus require **scope-prefixed instance names** for tenant-scoped helper DOs, with the **wall being `onBeforeCall` rejection** (not a naming helper). This closes: helper cross-tenant access (pre-claim + guess), Star pre-claim, and shared-Galaxy/Universe multi-star rejection.

## Background ‚Äî what we found (verified in code)

`NebulaDO.onBeforeCall` ([nebula-do.ts:30-46](../apps/nebula/src/nebula-do.ts#L30)) stores the **first** caller's `aud` in `__nebula_universeGalaxyStarId` and rejects later mismatches; it **never compares `aud` to `this.lmz.instanceName`**.

**`onBeforeCall` is the SOLE tenant gate on the mesh hot path ‚Äî not a redundant backstop.** The Gateway forwards the **client-chosen `(binding, instance)` verbatim** ‚Äî `NebulaClientGateway` overrides only `onBeforeCallToClient` (the push direction), there is no inbound scope check ([nebula-client-gateway.ts:13](../apps/nebula/src/nebula-client-gateway.ts#L13)), and `#handleClientCall` routes to `getDOStub(env[binding], instance)` with only `aud` pinned to the verified WS attachment. The HTTP-path `matchAccess` ([router.ts:382](../packages/nebula-auth/src/router.ts#L382)) does **not** run on per-call mesh dispatch. So a logged-in `aud=acme.app.tenant-a` client can issue `lmz.call('STAR','<victim>')` directly; only `onBeforeCall` stops it. (A foreign **DO** can do the same DO‚ÜíDO ‚Äî which is why the helper threat can't be solved at the Gateway; only the callee's own `onBeforeCall` covers both origins.)

Three consequences of TOFU-not-structural:

1. **Helper DOs.** A `NebulaDO` helper named by anything other than its scope (a `resourceId`, a local string) loses the instance-name‚Üîscope coupling. Isolation degrades to TOFU (race-able *before* first claim) + name-unguessability. A guessable / enumerable / caller-influenced name lets a different tenant **pre-claim** it or call it cross-tenant.
   - **NOT "unwired."** `ResourceHistory` is wired across the **test** surface with a bare `generateUuid()` resourceId ([baseline/index.ts:249](../apps/nebula/test/test-apps/baseline/index.ts#L249)), and [`scope-binding.test.ts`](../apps/nebula/test/test-apps/baseline/scope-binding.test.ts) **ossifies the exact TOFU semantics this task removes** (star/galaxy/universe binding + "rejects cross-scope"). That suite must be **rewritten**, not kept. A bare UUID also mis-tiers: `parseId(uuid)` reads a 1-segment "universe" slug ‚Üí derives `uuid.*` (wrong), it does **not** throw.
2. **Star pre-claim.** Because `onBeforeCall` never checks `aud`-vs-name, an attacker addressing `STAR/<victim>` with their own `aud` *before the victim's first login* sets the TOFU lock to the wrong scope. (Severity-bounded ‚Äî see Fix 1 / M-note below.)
3. **Shared Galaxy/Universe multi-star.** `Galaxy extends NebulaDO` (no override); it's called both by Stars with their inherited **star-level** `aud` ([star.ts:384-386](../apps/nebula/src/star.ts#L384)) and by galaxy-admins with **galaxy-level** `aud` ([scope-binding.test.ts:94](../apps/nebula/test/test-apps/baseline/scope-binding.test.ts#L94)). TOFU locks to whichever `aud`-level arrives **first** and rejects the rest ‚Äî so the galaxy-shared ontology can't serve a second star (or mixes star/galaxy levels inconsistently). **Latent:** tests use a unique universe per run, masking it.

Root: isolation rests on first-use luck + name-unguessability, not structure.

## Design

### Fix 1 ‚Äî Structural, tier-aware `onBeforeCall`
Accept iff `aud` is covered by the scope encoded in the instance name, reusing the existing id-algebra:

```
const name = this.lmz.instanceName
if (!name) throw 'Mesh call missing callee instance name'      // fail-closed (M7): envelope lacked metadata.callee
const scope = scopeFromInstanceName(name, this.lmz.bindingName) // SINGLE shared derivation (m-1) ‚Äî binding-aware, hard-rejects malformed (Fix 2); no inline SEP split (n-6)
if (scope === PLATFORM_INSTANCE_NAME) throw 'Active-scope mismatch'  // M-1: buildAuthScopePattern('nebula-platform') === '*' (accept-all); no tenant DO may bear it
const pattern = buildAuthScopePattern(scope)                    // throws if scope unparseable ‚Üí fail-closed
const aud = this.lmz.callContext.originAuth?.claims?.aud
if (!aud) throw 'Missing active scope (aud)'                    // unchanged
if (!matchAccess(pattern, aud)) throw 'Active-scope mismatch'
```
- **Star** (name = star id) ‚Üí exact pattern ‚Üí `aud` must equal the star.
- **Galaxy / Universe** (name = galaxy / universe id) ‚Üí `‚Ä¶‚Äã.*` ‚Üí accepts the scope itself **and any descendant**, rejecting other tenants. This is the intended "a Star reaches its own Galaxy/Universe (and they it), never another's" model (**M1 resolved**). **Authorization stays per-method:** the scope check is the *tenant boundary*; whether a given call must be admin-originated is decided by `@mesh(requireAdmin)` as today.
  - **Phase-1 audit (M1):** enumerate every **non-admin** `@mesh` method on `Galaxy`/`Universe` and confirm each holds galaxy/universe-**shared** data any descendant star may read; move any star-specific one behind `requireAdmin` or a per-star check. Current surface (`grep -n '@mesh()' apps/nebula/src/{galaxy,universe}.ts`): **Galaxy** `getGalaxyConfig` ([:90](../apps/nebula/src/galaxy.ts#L90)), `getLatestOntologyVersion` (:128), `getOntologyVersion` (:139), `listOntologyVersions` (:145); **Universe** `getUniverseConfig` ([:17](../apps/nebula/src/universe.ts#L17)) ‚Äî **five, not one**. `getLatestOntologyVersion` is load-bearing: it's the method Stars call cross-tier with a **star-level** `aud` ([star.ts:384-386](../apps/nebula/src/star.ts#L384)), so its "shared, any descendant may read" classification is what proves the `.*` widening sound. **Ongoing invariant** (not a one-time check): every new non-admin `@mesh` method on Galaxy/Universe must be shared-tenant data.
- **Drop the `__nebula_universeGalaxyStarId` store.** Confirmed single writer/reader (nebula-do.ts:40-42) ‚Äî code-safe to remove.
- **One derivation, not two (m-1).** `onBeforeCall` derives `scope` via the **same** `scopeFromInstanceName(name, binding)` used to mint/parse helper names (Fix 2, in `@lumenize/routing`) ‚Äî never an inline `indexOf(':')`. The sole tenant boundary must not be computed by two code paths that can drift.
- **Platform-name guard (M-1).** `buildAuthScopePattern('nebula-platform')` returns `'*'`, and `matchAccess('*', anyAud)` is always true ‚Äî so a tenant DO bearing the platform instance name (or a helper whose `:`-prefix parses to it) would collapse the gate to accept-all. None of Universe/Galaxy/Star/ResourceHistory is the platform DO, so `onBeforeCall` hard-rejects `scope === PLATFORM_INSTANCE_NAME` and `scopedName` rejects a platform prefix. Covered by **T-platform**.
- **Why this is sound (M4 ‚Äî it *relocates* TOFU, doesn't "kill" it).** `this.lmz.instanceName` is itself first-write-wins (`setInstanceName`, [lmz-api.ts:588](../packages/mesh/src/lmz-api.ts#L588)). The invariant (m-2): a NebulaDO is **addressed by the same `parseId`-valid slug/scoped-name that becomes its `instanceName`** ‚Äî never a 64-char-hex DO ID, since `scopedName` rejects `:`/`.` and slugs aren't 64-hex, so `getDOStub`'s `idFromString` branch ([get-do-stub.ts:30-37](../packages/routing/src/get-do-stub.ts#L30)) is unreachable for these names ‚Äî hence **derived scope == routing key**. So an attacker can only write a name into the DO *at that name's address*; they can't make a victim's DO adopt a foreign name, and `matchAccess(buildAuthScopePattern(name), attackerAud)` fails closed. Pre-claim degrades to **benign DoS-by-naming**, not a hijack.
- **Stamping invariant (m-3).** Soundness also requires `metadata.callee.instanceName` === the routing key passed to `getDOStub` on **every** path reaching a NebulaDO (Gateway `#handleClientCall`; `callRawImpl`). `__init` persists it first-write-wins, then `onBeforeCall` reads it ‚Äî name and address are the same string by construction. Add a Phase-3 defense-in-depth assertion that fails if a DO's first-write `instanceName` diverges from its stub address.
- **`instanceName` is readable here (M7).** `__init` runs from envelope metadata *before* `onBeforeCall` ([lmz-api.ts:877-895](../packages/mesh/src/lmz-api.ts#L877)) and persists ‚Äî but only when `metadata.callee.instanceName` is present; absent ‚áí the fail-closed throw above. (Note: the alarm-delivered and OCAN self-continuation paths run via the *local* executor, not `executeEnvelope`, so this throw never fires on them.)
- **Dependency (M5) ‚Äî resolved: value-import, no relocation (m-6).** This adds nebula-do.ts's first **value** import from `@lumenize/nebula-auth` (currently type-only). Legal (nebula‚Üínebula-auth; no cycle) ‚Äî **keep it.** Do **not** relocate `parseId`/`buildAuthScopePattern` to `@lumenize/routing`: they encode Nebula tier/slug vocabulary (and transitively pull `PLATFORM_INSTANCE_NAME` + `ParsedId`/`Tier`), and `routing` is deliberately Nebula-agnostic (per mesh.md dependency-direction; its only runtime dep is `@lumenize/debug`). If a home for the pure matcher is ever wanted, relocate **only** `matchAccess` (the one tier-agnostic string op) ‚Äî but not in this task.

### Fix 2 ‚Äî Scope-prefixed helper names; the wall is `onBeforeCall`, not the helper (B5)
- **The wall** is Fix 1 rejecting any tenant-scoped-helper call whose instance name isn't a well-formed `<scope>:<localName>` with `<scope>` covering `aud`. A bare `resourceId` is rejected loudly ‚Äî proven by a test calling the helper with a raw UUID and asserting rejection. `scopedName()` is **ergonomics on top**, not the enforcement.
- **Grammar (B4/M3), pin before "go":** separator `:` (illegal in a slug, so `parseId` rejects it = the loud-failure mechanism). `scopedName(localName)` rejects any `localName` containing `:` or `.` (slug charset only) and validates the prefix via `parseId`. `scopeFromInstanceName` splits on the **first** `:`, `parseId`-validates the left side, and **hard-rejects** (no whole-name fallback) when a helper binding's name has no `:` or an unparseable prefix. Make it **binding-aware**: declare the set of tenant-scoped-helper bindings (`RESOURCE_HISTORY`, future) that REQUIRE the `:` form; tier bindings (`STAR`/`GALAXY`/`UNIVERSE`) require a bare id.
- **`scopedName` placement (M6):** pure form in `@lumenize/routing` ‚Äî `scopedName(scope, localName)` (reusable + testable; usable from the Gateway/client which have no `this.lmz`), plus a thin `NebulaDO.scopedName(localName)` delegating via `this.lmz.instanceName`.
- **Worked example (m4/m5):** `ResourceHistory` keyed `acme.app.tenant:<id>` instead of `<uuid>`; migrate [baseline/index.ts:249](../apps/nebula/test/test-apps/baseline/index.ts#L249) to mint it via `scopedName`; rewrite the `resource-history.ts` docstring (it currently re-teaches the removed TOFU model).
- **Scope (m-7) ‚Äî RESOLVED (2026-06-08): build the full grammar now.** `ResourceHistory` has no production caller *today* (sole invoker is the test initiator `callResourceHistoryGetHistory`), so the `:`-grammar is, for the moment, driven by a fixture. **Decision:** ship Fix 2 in full anyway ‚Äî `scopeFromInstanceName` + the binding-aware registry + public `scopedName()` + migrate `ResourceHistory` to `<scope>:<id>`. **Rationale update (2026-06-08, later same day):** the original "because wiring `ResourceHistory` for the Resources work is imminent" no longer holds ‚Äî resource history moved to **R2**, so `ResourceHistory` will *not* become a real history-storage helper (see `tasks/on-hold/nebula-resource-history-r2.md`). The decision is unchanged: build the grammar now because (a) `ResourceHistory` persists as the canonical tenant-scoped-helper **fixture** and (b) the `:`-naming primitive serves *any* future tenant-scoped helper, not this one specifically. Design `scopedName` against that general use, not the fixture's incidentals. Phase 2 + T4/T4b/T4-race ship as written.

### Migration ‚Äî hard-switch, NOT compare-and-reject (M2)
The stored `__nebula_universeGalaxyStarId` is **whatever `aud` first claimed the DO** (nebula-do.ts:40-42 stores the first caller's aud verbatim). For a Galaxy that can be a star-level descendant (a Star's hop) **or** a galaxy-level value (a galaxy-admin's first call, [scope-binding.test.ts:99-103](../apps/nebula/test/test-apps/baseline/scope-binding.test.ts#L99)) ‚Äî depending on who logged in first. It is therefore **per-first-caller and tier-ambiguous**, hence structurally untrustworthy: "read-through-then-reject-on-mismatch" would both lock out legitimately-claimed instances and rest on a value that doesn't reliably encode tier. **Trust the instance name; ignore and drop the stored value** ‚Äî a one-time, **conditional** delete (read once, delete only if present, so it is *not* a per-call write). Don't re-add a compare later.

### Out of scope (with pointers)
- **Raw Workers-RPC bypass of `@mesh`/`onBeforeCall` (B2).** A direct `stub.method()` skips `executeEnvelope` entirely, so no decorator/`onBeforeCall` guard runs ‚Äî for **any** public method, including `@mesh(requireAdmin)`. Today's protection is the "no raw RPC" convention (`mesh.md`). ‚öÝÔ∏è **Now that `onBeforeCall` is the *sole* structural tenant wall (m-8), be precise about the claim:** "structural" means structural **for all mesh-routed calls**; raw RPC remains a **known unenforced gap** covered only by the no-raw-RPC convention. In-task action: a Phase-3 audit to `#`-privatize NebulaDO methods that are *only* called internally (OCAN continuation targets like `doSubscribe` must stay public ‚Äî document them as a known raw-RPC surface). The full *structural* enforcement (an `@mesh` wrapper requiring an active `callContext`) is its own follow-on, sequenced after this ‚Üí [mesh-active-callcontext-guard.md](mesh-active-callcontext-guard.md).
- **Non-mesh `fetch` routes** on tenant-scoped DOs ‚Äî none today; keep it that way (audit below).
- **Gateway-side inbound scope check** ‚Äî rejected: `onBeforeCallToMesh` covers only `NebulaClient`‚Üímesh, not the DO‚ÜíDO helper threat, which Fix 1 already covers on both origins.

## Test matrix (capable-of-failing ‚Äî each flips if reverted to TOFU)
- **T1 ‚Äî Shared-Galaxy multi-star** (RED‚ÜíGREEN): two sibling stars (galaxy-level login, distinct `activeScope`) both read the shared ontology. Fails on TOFU (second star rejected).
- **T2 ‚Äî Cross-galaxy isolation, capable against the *widening*** (M9): pair "sibling star under same galaxy IS accepted" (positive, = T1) with "a galaxy-X-scoped admin calling `GALAXY/Y` is rejected" via a **real second client**, not a hand-crafted aud ‚Äî so it exercises Fix 1's `matchAccess`, not a tautology.
- **T3 ‚Äî Star pre-claim** (RED‚ÜíGREEN) (M8): attacker client (own scope) invokes a `NebulaClientTest` initiator with `starInstanceName=<victim>` decoupled from its `aud`, as the **first** call to a fresh victim Star. RED = succeeds + pre-claims; GREEN = "Active-scope mismatch". Assert the victim's later legitimate first call still succeeds (no residual lockout, M4).
- **T4 ‚Äî Helper cross-tenant**: tenant B cannot reach tenant A's `<A-scope>:foo`; two tenants' same `localName` ‚Üí different instances.
- **T4b ‚Äî Helper grammar** (B4): well-formed `<scope>:<local>` with matching `aud` resolves; malformed/unscoped (`foo`, `acme.app.tenant:`, `a.b.c:bad:extra`, crafted `localName` with `:`/`.`) rejected loudly ‚Äî and a crafted `localName` **cannot shift** the derived scope. **Also (m-9):** a *tier* binding (e.g. `STAR`) addressed with a `parseId`-rejecting name fails closed (`buildAuthScopePattern` throws) as an observable client `lastError`, not a swallowed 500 ‚Äî covers Fix 1's fail-closed branch (b).
- **T4-race ‚Äî Reject is name-derived, not order-derived** (M11): run **both** orderings (B-then-A and A-then-B) at the same fresh `<A>:foo`; A always succeeds, B always rejects ‚Äî **identical outcomes regardless of order**. Input-gate serialization means this guards a *TOFU-reintroduction* regression (a store-on-first-claim gate would let B-first lock out A), not a live data race. Flips on TOFU (order-dependent).
- **T5 ‚Äî Missing aud still rejected** (system/no-aud, and envelope-missing-callee per M7), unchanged.
- **T6 ‚Äî Tier-mismatch** (m1/m2): a legitimately-minted galaxy-scoped admin (`aud = acme.app`) calling a Star is rejected at the Star's `onBeforeCall` ‚Äî resolves the (now-settled) galaxy-admin-aud question; positive control = [scope-binding.test.ts:60](../apps/nebula/test/test-apps/baseline/scope-binding.test.ts#L60) (wildcard admin with `activeScope=star` ‚Üí `aud=star` ‚Üí accepted).
- **T-platform ‚Äî Platform-name not an any-aud sink** (M-1): a tenant-scoped DO addressed at instance name `nebula-platform` (and a helper `nebula-platform:foo`) is **rejected**, not accepted, for a foreign `aud`. Flips if the `scope === PLATFORM_INSTANCE_NAME` guard is removed (gate degrades to `matchAccess('*', aud)` = always true).
- **T-migration** (M10): seed `__nebula_universeGalaxyStarId` to a stale/wrong scope (test initiator or `ctx.storage.kv.put`); assert the name-derived check **wins** over the stale value.
- Per-`it` unique scope ids (m3) ‚Äî claim-sensitive tests must not share a Galaxy/helper id across `it` blocks (`isolatedStorage:false` for WS suites).
