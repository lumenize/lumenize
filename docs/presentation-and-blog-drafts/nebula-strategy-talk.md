## Spine (from `strategy.md`)

- One line: **"Nebula is where domain experts build secure SaaS apps and get paid."**
- *Why now*: the tools ship apps wide open — no auth, no access control, an agentic layer nobody has secured (CVE-2025-48757, EchoLeak). The cost just went up (strict liability, machine-speed attacks).
- The fused claim: **secure *and* agentic — one claim, not two.** The same access substrate that secures the app governs what the AI can answer.
- One product, two buyers, sequenced: self-serve solopreneur/intrapreneur wedge now → enterprise expansion later (the shadow-IT → DevSecOps arc; see `enterprise.md`).

### The iron triangle of agentic development

**Canonical write-up:** [`the-iron-triangle-of-agentic-development.md`](the-iron-triangle-of-agentic-development.md) — the argument, the CVEs, the rail-network reframe, and the open naming question (Swift vs Fearless) all live there. **Edit there, not here.**

- **Technique:** raise the strongest objection to your own thesis *out loud*, concede it fully, *then* flip it. The flip only lands if the room first believes the objection is fatal.
- **Beat sheet (spoken):**
  1. **Concede** — "Lock the AI to only what the user can see → worse answers. Security costs you quality. Right?"
  2. **The triangle** — Secure · Smart · Swift, choose 2; walk the three rows the room has lived (dumb / leaky / molasses).
  3. **The flip** — never physics, just a broken *model*: not only coarse but *rigid* — real orgs aren't trees (the Fortune-50 10k-node, multi-parent example); two moves (multi-parent DAG + JIT elevation) → choose 3.
  4. **Land it** — reject "guardrail" (by the time you touch it you've already left the road) → the rail-network image: can't derail, reaches everywhere, new track laid on demand.
